Q2: How did you approach troubleshooting and debugging any broken links or references?

When troubleshooting broken links or references, I follow these steps: Check the Path: Make sure the file or link path is correct. Even small mistakes, like wrong capitalization or missing file extensions, can cause issues. Test the Link: Open the link directly in the browser to see if it works. Use developer tools to check if the file is missing or causes an error. Verify File Names: Double-check that the file names match exactly, including capitalization and extensions (e.g., .jpg vs .jpeg). Check Permissions: Make sure the files have the right permissions to be accessed. Test in Different Browsers: Sometimes, broken links only show up in specific browsers. Test in multiple browsers or use incognito mode to avoid caching issues. Use Link Checkers: Use online tools like W3C Link Checker to find broken links. Clear Cache: Clear the browser cache or do a hard refresh (Ctrl + F5) to load the latest version.
